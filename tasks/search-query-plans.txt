=== 5. bitboard prime pattern pre-filter ===

QUERY PLAN
|--SCAN p
`--SEARCH a USING INDEX idx_analysis_position (position_id=?) LEFT-JOIN

Note: SCAN is expected — bitwise AND expressions cannot use B-tree indexes in
SQLite. The pre-filter value is crucial: these two AND conditions together rule
out ~98% of rows cheaply (integer arithmetic, no I/O), so the effective cost is
a fast sequential scan of the integer columns only (state JSON is not read until
the row passes both masks).
